# Core Intent Analysis Mechanism

## 核心理念
本产品的核心是对用户浏览意图的精准剖析、理解和抽象。任务-子任务-页签-子页签-孙页签是可视化的基本结构。
自动化的机制以及 AI 分析的划分机制是本产品的最核心部分。

## 机制定义
1. **意图识别 (Intent Recognition):**
   - 利用 AI 分析页面内容（不仅仅是域名），识别用户的真实浏览意图（例如区分 "Debug" vs "Learning"）。
   - 将浏览行为转化为具象的任务流。

2. **自动分组 (Automatic Grouping):**
   - 根据识别出的意图，将相关的标签页自动聚合。
   - 形成清晰的 "Task Flow"（任务流）。

3. **可视化 (Visualization):**
   - 将混乱的浏览历史和当前标签页转化为可视化的思维导图/树状结构。
   - 清晰呈现 "完成了什么" 和 "思考路径"。

4. **数据沉淀 (Knowledge Settlement):**
   - 支持将任务流导出 (Markdown/PDF)，形成知识复盘。

## 维护原则
- **稳定性:** 任何修改都必须保证核心能力的畅通。
- **系统性:** 修改时需从本源思考，避免治标不治本。需考虑机制间的交错影响。
- **一致性:** 确保 AI 分析的准确性和插件功能的连通性。
